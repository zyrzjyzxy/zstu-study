**2****．****实现图像直方图均衡化（不能直接函数调用）**

图像直方图（Image Histogram）是用于表示图像中亮度分布的图形，描述了 图像中每个亮度值的像素数。在直方图中，横坐标的左侧为较暗的区域，横坐标 的右侧为较亮的区域。因此，如果是一张较暗的图像，它的直方图的数据多集中 在左侧和中间的部分；而整体明亮、只有少量阴影的图像则相反，多集中在中间和右侧的部分。

直方图均衡化就是把原始图像的灰度直方图从比较集中的某个灰度区间变 成在全部灰度范围内的均衡分布，它对图像进行非线性拉升，重新分布图像像素 值，使一定灰度范围内的像素数量大致相同，也就是把给定图像的直方图分布改变成“均匀 ”分布直方图分布



### 直方图均衡化算法第一步：统计直方图每个灰度级出现的次数



直方图均衡化算法包含三个步骤，此为第一步。具体实现代码及原理如下：



代码部分：



```c
// 外层循环遍历图像的每一行，height为图像高度
for(i = 0; i < height; i++) { 
    // 内层循环遍历图像的每一列，width为图像宽度
    for(j = 0; j < width; j++) { 
        // n数组记录每个灰度级出现的次数，s[i][j]为坐标(i, j)处像素的灰度值
        n[s[i][j]]++; 
    }
}
// 遍历所有灰度级，L为最大灰度级
for(i = 0; i <= L; i++) { 
    // 计算每个灰度级在整幅图像中出现的概率，存储在p数组
    p[i] = n[i]/(width*height); 
}
```



变量说明：



- `n[i]`：表示灰度级为`i`的像素的个数。
- `L`：表示最大灰度级 。
- `width`和`height`：分别表示原始图像的宽度和高度。
- `p[i]`：表示灰度级为`i`的像素在整幅图像中出现的概率，即存储了这幅图像归一化之后的直方图。

### 直方图均衡化算法第二步：累计归一化的直方图



1. **代码实现**



```c
for(i = 0; i <= L; i++) {
    for(j = 0; j <= i; j++) {
        c[i] += p[j];
    }
}
```



1. **说明**：通过两层循环，将灰度级从起始到当前`i`的概率`p[j]`进行累加，存储在`c[i]`中 。`c[]`数组存储的就是累计的归一化直方图。

### 直方图均衡化算法第三步：计算新的像素值



1. 找出像素的最大值和最小值
   - **代码实现**



```c
max = min = s[0][0];
for(i = 0; i < height; i++) {
    for(j = 0; j < width; j++) {
        if(max < s[i][j]) {
            max = s[i][j];
        } else if(min > s[i][j]) {
            min = s[i][j];
        }
    }
}
```



- **说明**：通过两层循环遍历图像每个像素`s[i][j]`，找出其中的最大值`max`和最小值`min` 。



1. 计算最终均衡化结果
   - **代码实现**





```c
for(i = 0; i < height; i++) {
    for(j = 0; j < width; j++) {
        t[i][j] = c[s[i][j]] * (max - min) + min;
    }
}
```



- **说明**：再次遍历图像每个像素，根据累计归一化直方图`c[]`以及前面找出的像素最大最小值，通过公式计算出最终直方图均衡化之后的像素值，存储在`t[][]`中。

### 实验代码

```python
import numpy as np
from PIL import Image
import matplotlib.pyplot as plt
import cv2

# 读取图像并转换为灰度图
def read_image(path):
    img = Image.open(path).convert('L')
    img_array = np.array(img)
    return img_array

# 自定义直方图均衡化（基于实验步骤）
def custom_hist_equalize(img_array):
    height, width = img_array.shape
    # 统计灰度级出现次数
    n = np.bincount(img_array.ravel(), minlength=256)
    p = n / (height * width)  # 计算概率
    # 计算累积直方图
    c = np.cumsum(p)
    # 计算新像素值
    min_val, max_val = img_array.min(), img_array.max()
    equ_array = (c[img_array] * (max_val - min_val) + min_val).astype(np.uint8)
    return equ_array

# 对比：OpenCV直方图均衡化
def cv2_hist_equalize(img_array):
    return cv2.equalizeHist(img_array)

# 绘制直方图对比
def plot_hist_comparison(original, custom, cv2_eq, filename):
    plt.figure(figsize=(15, 10))
    
    # 原图
    plt.subplot(3, 2, 1)
    plt.imshow(original, cmap='gray')
    plt.title('Original Image')
    plt.axis('off')
    
    plt.subplot(3, 2, 2)
    plt.hist(original.ravel(), bins=256, range=(0, 255), density=True)
    plt.title('Original Histogram')
    
    # 自定义均衡化
    plt.subplot(3, 2, 3)
    plt.imshow(custom, cmap='gray')
    plt.title('Custom Equalized Image')
    plt.axis('off')
    
    plt.subplot(3, 2, 4)
    plt.hist(custom.ravel(), bins=256, range=(0, 255), density=True)
    plt.title('Custom Equalized Histogram')
    
    # OpenCV均衡化
    plt.subplot(3, 2, 5)
    plt.imshow(cv2_eq, cmap='gray')
    plt.title('OpenCV Equalized Image')
    plt.axis('off')
    
    plt.subplot(3, 2, 6)
    plt.hist(cv2_eq.ravel(), bins=256, range=(0, 255), density=True)
    plt.title('OpenCV Equalized Histogram')
    
    plt.tight_layout()
    plt.savefig(filename)
    plt.close()

# 主函数
def main():
    img_array = read_image('input.jpg')
    custom_eq = custom_hist_equalize(img_array)
    cv2_eq = cv2_hist_equalize(img_array)
    
    # 保存结果
    Image.fromarray(custom_eq).save('output_custom.jpg')
    Image.fromarray(cv2_eq).save('output_cv2.jpg')
    
    # 生成对比图
    plot_hist_comparison(img_array, custom_eq, cv2_eq, 'hist_comparison.png')

if __name__ == '__main__':
    main()
```

### 实验分析

#### 1. 图像读写操作
• 使用 `PIL.Image` 读取图像并转换为灰度格式，确保后续处理基于单通道数据。
• 通过 `numpy` 将图像转换为数组，便于高效处理像素数据。

#### 2. 直方图均衡化实现
• **步骤1**：统计每个灰度级出现的次数，利用 `np.bincount` 快速计算。
• **步骤2**：计算累积直方图，通过 `np.cumsum` 实现概率累积。
• **步骤3**：根据实验提供的公式，将原图灰度值映射到其原始动态范围（`min` 到 `max`）内，保留原始对比度基础。

#### 3. 结果评估
• **对比分析**：通过对比自定义算法与OpenCV的结果，发现：
  • **动态范围**：自定义算法保留原图动态范围，而OpenCV将结果扩展到0-255，整体对比度更高。
  • **直方图分布**：自定义算法的直方图在原范围内更均匀，OpenCV的分布覆盖全范围且更平坦。
  • **视觉效果**：OpenCV结果可能因对比度拉伸过度丢失细节，自定义算法在保留原图亮度分布的同时增强局部对比度。

#### 关键区别
• **动态范围处理**：自定义算法基于原图范围调整，适合保留原始亮度特征；OpenCV方法最大化对比度，适用于全局增强。
• **适用场景**：若需保持图像自然亮度（如医学影像），自定义方法更优；若需最大化信息可见性（如卫星图像），则OpenCV方法更佳。
